### 1. 编译程序和解释程序
```c
解释器：翻译源程序时不生产独立的目标程序（解释程序和源程序都要参与运行过程中）
编译器：生成独立保存的目标程序（不在参与目标程序的运行过程、机器上运行的是源程序等价的目标程序）
```

### 2. 数据类型
```c

便于规定数据对象的取值范围及能够进行的运算
```


### 4. 翻译阶段、符号表
```c
编译方式：词法分析、语法分析、语义分析、中间代码生成、代码优化、目标代码
解释方式：词法分析、语法分析、语义分析（参与运行控制，执行速度慢）

编译器和解释器都不可省略前3个且顺序不可交换
编译器方式中中间代码生成和代码优化不是必要，可省略
即编译器方式可以在词法分析、语法分析、语义分析阶段后直接生成目标代码
----------------------
符号表：
不断收集、记录和使用源程序中一些相关符号的类型和特征等信息，并将其存入符号表中
记录源程序中各个字符的必要信息，以辅助语义的正确性检查和代码生成
----------------------
1.词法分析：
输入：源程序  输出：记号流
分析构成程序的 (字符) 及由字符按照构造规则构成的符号是否符合程序语言的规定

2.语法分析输入：记号流  输出：语法树
可以发现程序中所有的 (语法错误)
对各条语句的 (结构) 进行合法性分析
分析程序中的句子结构是否正确

3.语义分析输入：语法树
语义分析阶段的主要作用是进行 (类型) 分析和检查

语义分析阶段不能发现程序中所有的语义错误
语义分析阶段可以发现 (静态语义错误)
不能发现动态语义错误，动态语义错误运行时才能发现

4.目标代码生成目标代码生成阶段的工作与 (具体的机器) 密切相关
(寄存器) 的分配工作处于目标代码生成阶段

5.常见中间代码：后缀式、三地址码、三元式、四元式和树图等形式
中间代码与具体的机器无关 (不依赖具体的机器)，
可以将不同的高级程序语言翻译成同一种中间代码中间代码可以 (跨平台)
因为 与具体的机器无关，使用中间代码有利于进行 (与机器无关的优化处理和提高编译程序的可移植性)
```

### 6. 有限自动机
```c
有限自动机是 (词法分析的一个工具) ，它能正确地识别正规集

确定的有限自动机(DFA)：对每一个状态来说识别字符后转移的状态是唯一的
不确定的有限自动机 (NFA)：对每一个状态来说别字符后转移的状态是不唯一的
----------------
一个圈为初态、两个圈为既可以为初态也可为终态
完事后需要停留在终态
```

### 8. 中缀后缀、中序后序转换
